com.google.xml.combinators

XmlInputStore

trait XmlInputStore extends XmlStore

This class represents the input of picklers. It keeps around XML attributes, nodes and current namespace bindings. There are two modes in which input stores operate: linear and random access. A linear store will return elements in the order in which they are defined in the XML document. A random access store will look the element up anywhere in the 'nodes' collection. Switching between the two modes is done by calling 'randomAccessMode' and 'linearAccessMode'. The modes nest (randomAccessMode and linearAccessMode calls have to match).

All 'accept' methods return an instance of XmlInputStore which is the input after the accepted element was consumed. Implementers may choose whether to implement it using mutable state (and return 'this') or use an immutable representation.

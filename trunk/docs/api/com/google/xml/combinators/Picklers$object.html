object Picklers
extends TupleToPairFunctions
A class for XML Pickling combinators.

A pickler for some type A is a class that can save objects of type A to XML (pickle) and read XML back to objects of type A (unpickle). This class provides some basic building blocks (like text), and several combinators (like elem, attr, seq) to build more complex picklers.

Example:

def picklePair: Pickler[String ~ String] = elem("p", URI, "pair", elem("p", URI, "a", text) ~ elem("p", URI, "b", text)) val input = <p:pair xmlns:p="testing-uri"> <p:a>alfa</p:a> <p:b>omega</p:b> </p:pair> picklePair will be able to pickle and unpickle pairs of Strings that look like the input.

Transform the given parser into a parser that accepts permutations of its containing sequences. That is, interleaved(a ~ b ~ c) will parse a, b, c in any order (with possibly other elements in between. It should not be called directly, instead use the interleaved which wraps an element around the interleaved elements.
def interleaved [A](label : java.lang.String, pa : => Pickler[A])(implicit ns : (java.lang.String, java.lang.String)) : Pickler[A]
Convenience method for creating an element with interleaved elements. Elements enclosed by the given element label can be parsed in any order. Any unknown elements are ignored.

Example: interleaved("entry", elem("link", text) ~ elem("author", text)) will will parse an element entry with two subelements, link and author, in any order, with possibly other elements between them.
def list [A](sep : Char, pa : => Pickler[A]) : Pickler[scala.List[A]]
Pickler for a list of elements. It unpickles a list of elements separated by 'sep'. It makes little sense to use this pickler on elem or attr picklers (use '~' and 'rep' instead.) For an example how this is used, see MediaRss which defines a comma-separated list of categories as the contents of an element.

def when [A, B](pa : => Pickler[A], pb : => Pickler[B]) : Pickler[B]
Runs 'pb' unpickler on the first element that 'pa' successfully parses. It is more general than 'interleaved', which uses only the element name to decide the input on which to run a pickler. 'pa' can be arbitrarily complex. Example: when(elem("feedLink", const(attr("rel", "#kinds"), rel)), kindsPickler) will look for the first 'feedLink' element with an attribute equal to '#kinds' and then run 'kindsPickler' on that element.

Class Summary
case class Error (val m : java.lang.String, val i : XmlInputStore) extends NoSuccess with scala.Product
An Error is a failure which causes the entire parsing to fail (no alternatives are tried).
case class Failure (val m : java.lang.String, val i : XmlInputStore) extends NoSuccess with scala.Product
A Failure means the parsing has failed, but alternatives can still be tried.
abstract class NoSuccess (val msg : java.lang.String, val in : XmlInputStore) extends PicklerResult[Nothing]
Parsing failed. There are two possible subclasses, Failure and Error. Failure is recoverable, while Error makes the whole parsing fail. To make a pickler return errors, apply 'commit' to it. All 'elem' picklers will commit on their contents, so that an error parsing the contents of an element will stop everything. This is almost always the best thing to do (and yields the best error messages).
